Portfolio — IT 312

Hey, I’m Amiir Mitchell.

I’m an Information Technology student at Central Washington University with a concentration in cybersecurity, currently working toward my Security+ and Network+ certifications. I also work on the ITS help desk, which means I spend a lot of time watching real people get stuck in real interfaces — and that’s exactly what pulled me into this course. Almost every ticket I take is a design problem wearing a technical costume: a confusing error message, a buried setting, a system that assumes knowledge nobody actually has. Security taught me the same lesson from a different angle — the strongest control in the world fails if it’s too frustrating for people to use correctly. So I’ve stopped asking why users get things wrong and started asking why systems make it so easy to. This portfolio is a quarter spent learning how to design ones that don’t.

The work

This portfolio collects everything I built this quarter, and it traces an arc: the blog starts with systems thinking — how complexity moves through a system and where failures actually live — then works down into visual perception, memory, and the psychology behind UX law, ending with the ethics of using that psychology responsibly. Every post was written by hand in HTML and CSS and published through GitHub, so the blog isn’t just about design — it’s a working example of the development workflow we practiced all quarter. Alongside it are three design challenges built in Figma, moving from diagramming a digital system in its real-world context to a full interactive prototype built around a complete user journey. Nothing here is a first draft: each piece was revised through instructor feedback as the quarter went on, and honestly, the gap between the early versions and these is the clearest evidence of what I learned.

  • Blog Course Blog — Six Posts on Systems & UX Written reflections on systems thinking, life cycle models, perception, memory, and ethical interaction design.
  • Design Challenge 1 Context-Aware Digital System Diagrams A FigJam board mapping a digital system in its real context — components, interconnections, and purpose.
  • Design Challenge 2 Interface Design Prototype A Figma prototype applying visual hierarchy, contrast, and Gestalt grouping to interface layout.
  • Design Challenge 3 Interactive User Journey Prototype A clickable Figma prototype designed around a complete user journey, grounded in Hick’s, Jakob’s, and Fitts’s laws.
